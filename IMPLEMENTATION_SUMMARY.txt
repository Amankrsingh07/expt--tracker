================================================================================
                          HOW TO USE THE SYSTEM
================================================================================

STEP 1: SET BUDGETS
   1. Navigate to: http://localhost:3000/budget
   2. Click "➕ Set" on each category
   3. Enter budget amount (e.g., Food: 5000)
   4. Click "✓" to save
   5. Repeat for each category you want to limit

STEP 2: ADD EXPENSES
   1. Navigate to: http://localhost:3000/add-expense
   2. Enter amount and select category
   3. See real-time budget feedback:
      • Blue box = Within budget ✅
      • Yellow box = Approaching limit ⚠️
      • Red box = Exceeded budget ❌
   4. Click "Add Transaction" if allowed

STEP 3: MONITOR SPENDING
   1. Go to: http://localhost:3000/budget
   2. See all categories with:
      • Budget limit
      • Current spending
      • Progress bar
      • Remaining amount
   3. Red alerts show exceeded budgets
   4. Summary card shows overall stats

STEP 4: ADJUST BUDGETS
   1. Click "📝 Edit" on any category
   2. Enter new budget amount
   3. Click "✓" to save
   4. Or click "🗑️" to delete budget

================================================================================
                            COLOR SCHEME LEGEND
================================================================================

GREEN Progress Bar (0-80%):
   • Indicates safe spending zone
   • No warnings shown
   • Can add expenses freely

YELLOW Progress Bar (80-100%):
   • Indicates warning zone
   • Yellow alert box shown
   • "⚠️ Approaching limit!" message
   • Can still add expenses (with warning)

RED Progress Bar (>100%):
   • Indicates budget exceeded
   • Red card border
   • Red alert box
   • 🚨 animated icon
   • "❌ Budget Exceeded by ₹X" message

Blue Info Box:
   • Shows budget status during expense entry
   • Displays: Budget, Spent, Remaining, After amount
   • Normal state (no action needed)

Yellow Warning Box:
   • Shows when approaching limit (80-100%)
   • Allows submission (with caution)
   • Can still add the expense

Red Error Box:
   • Shows when budget would be exceeded (>100%)
   • Prevents submission
   • Submit button disabled
   • Must reduce amount to proceed

                              EXAMPLE WORKFLOW
================================================================================

SCENARIO: User wants Food budget of ₹5,000/month

Step 1: Set Budget
   → Go to /budget
   → Click "➕ Set" on 🍔 Food
   → Enter: 5000
   → Click "✓"
   → Card updates: "Budget: ₹5,000"

Step 2: Add First Expense (₹2,000)
   → Go to /add-expense
   → Amount: 2000
   → Category: Food
   → Blue info shows: Spent ₹0, Remaining ₹5,000, After ₹2,000
   → Click "Add Transaction" ✅
   → Success!

Step 3: Add Second Expense (₹2,500)
   → Go to /add-expense
   → Amount: 2500
   → Category: Food
   → Blue info shows: Spent ₹2,000, Remaining ₹3,000, After ₹4,500
   → Click "Add Transaction" ✅
   → Success!

Step 4: Try to Add Over Budget (₹2,000)
   → Go to /add-expense
   → Amount: 2000
   → Category: Food
   → Red error shows: "Budget Exceeded by ₹1,500"
   → Submit button DISABLED
   → Cannot add! ❌

Step 5: Reduce and Retry (₹500)
   → Change amount to: 500
   → Yellow warning shows: "Only ₹500 remaining"
   → Submit button ENABLED
   → Click "Add Transaction" ✅
   → Success!

Step 6: Check Budget Status
   → Go to /budget
   → Food card shows:
     • Progress bar at 100% (red)
     • Spent: ₹5,000
     • Remaining: ₹0
     • Red alert: "Budget Exceeded by ₹0" (at limit)

Step 7: Try Next Expense
   → Any amount > ₹0 will be blocked
   → Must delete or increase budget first

================================================================================
                              DEBUGGING TIPS
================================================================================

If budget enforcement not working:
1. Check browser console for JavaScript errors
2. Verify API endpoints are accessible (/api/category-budget)
3. Confirm category budget was saved (check database)
4. Check that expense is for "Expense" type (not Income)
5. Refresh page if state seems stale

If real-time validation not showing:
1. Wait 1-2 seconds for API response
2. Verify category is selected with budget set
3. Check network tab in DevTools for API calls
4. Ensure amount field has focus (user just typed)

If progress bar shows incorrect percentage:
1. Refresh the budget page
2. Verify all expenses are saved
3. Check that expenses match correct category
4. Manually calculate: (spent / budget) * 100

If submit button not disabling:
1. Hard refresh (Ctrl+Shift+R or Cmd+Shift+R)
2. Clear browser cache
3. Check for JavaScript errors in console
4. Try different browser
5. Check budget was saved correctly
